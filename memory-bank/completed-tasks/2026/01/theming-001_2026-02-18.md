## Задание

Двухслойная система тематизации (Color Scheme + Theme Variants):

- **Слой 1: Color Scheme** (светлый/тёмный режим)
  - Автоматическое определение системных настроек (`prefers-color-scheme`)
  - Ручное переключение: авто/светлая/тёмная
  - Базовые цветовые переменные (фон, текст, границы, тени)
- **Слой 2: Theme Variants** (цветовые темы)
  - Набор предустановленных тем (Ocean, Forest, Sunset, Lavender)
  - Каждая тема имеет light и dark варианты
  - Accent цвета и семантические цвета (success, warning, error)
- Техническая реализация: CSS Custom Properties, дизайн-токены, useTheme(), ThemeProvider
- UI: ThemeToggle, SettingsButton, SettingsModal, ThemeSelectionTab, HeaderControls
- Персистентность в localStorage, синхронизация между вкладками

## Изменения в задании

В UI нет явного выбора «авто» для Color Scheme. План предусматривал «авто / светлая / тёмная». Реализация: auto — дефолт при отсутствии сохранённых настроек; ThemeToggle переключает только light/dark; в SettingsModal draftScheme — light | dark. Пользователь не может вернуться к «авто» через интерфейс. Осознанное упрощение для MVP.

## Результат

Реализована двухслойная система тематизации: Color Scheme (светлый/тёмный/auto) и Theme Variants (Default, Ocean, Forest, Sunset, Lavender). ThemeProvider, useTheme, 5 тем × 2 схемы. ThemeToggle, SettingsModal, ThemeSelectionTab, HeaderControls. Рефакторинг SCSS на var(--color-\*). Сборка, линт, тесты — успешно.
