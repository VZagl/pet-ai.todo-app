## Description

Преобразование документации в `docs/` из описательного стиля в компактный императивный формат для оптимального восприятия ИИ.

**Проблема:**

Текущая документация содержит:

- Описательные формулировки ("Используй JSDoc для публичных функций" → должно быть "Добавлять JSDoc")
- Избыточные учебные примеры и "воду"
- Неконсистентную структуру между файлами
- Смешение учебного материала с правилами для ИИ
- Отсутствие явного указания, что это правила для ИИ (не для человека)
- Отсутствие правила о переносе выполненных задач из backlog в завершённые

**Цель:**

Создать компактную, императивную документацию с:

- Четким указанием "Правила для ИИ" в начале каждого файла
- Четкими командами вместо рекомендаций
- Минимальным набором критичных примеров
- Чеклистами ✅/❌ для проверки
- Единообразной структурой всех файлов

**Решение:**

1. Определить единый шаблон "Правила для ИИ" с заголовком назначения
2. Провести рефакторинг 9 файлов документации (git-workflow.md, git-commit-description.md, memory-bank-usage.md добавлены в процессе уточнения требований)
3. Убрать избыточные примеры, сохранив критичные
4. Преобразовать описания в императивы
5. Добавить чеклисты где уместно
6. Добавить в начало каждого файла заголовок с назначением
7. Добавить в memory-bank-usage.md правило о переносе выполненных задач из backlog
